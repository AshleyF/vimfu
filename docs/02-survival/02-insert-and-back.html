Insert and Back Again

i to type, Esc to come home

id: survival.insert-and-back · part: survival · keys: i, Esc, Ctrl-[ · lessons: 4, 5, 6, 15

Press i to enter Insert mode and type. Press Esc to return to Normal. Build the Esc reflex.

You are in Normal mode. The cursor sits on a character. Keys do things — they are commands. You want to type some text. How?

Press i. The status line shows -- INSERT --. Now keys do what you'd expect: they type characters.

Type whatever you want. Newlines work. Backspace works. This is just typing.

Coming back

Esc returns you to Normal mode. The status line empties. Keys are commands again. Ctrl-[ does the same thing — handy if your Esc key is far away.

Tip. Many remappers swap Caps Lock and Esc (or map Caps Lock to Esc). On many laptops, Esc is a tiny touch-strip key — moving it to Caps Lock is a real productivity win and easy to do in OS settings (no Vim config needed).

The Escape Habit

Build a reflex: after every thought, press Esc. Finished a sentence? Esc. Finished a function name? Esc. Done with that loop body? Esc. Normal mode is your resting state, not Insert. The minute you finish typing, get back to where you can move.

This sounds annoying. After a week, you stop noticing. After a month, you no longer remember what mode you are in — you just know which command to issue, and the right mode shows up.

🔧 Under the Hood — What Insert Mode Is Buffering

From the moment you press i until the moment you press Esc, Vim is recording the text you type into a small buffer. When you leave Insert, three things happen:

1. The text goes into the buffer (you see it). 2. The text is also stored in the ". register — accessible as the last-inserted-text register, useful in mappings and for the . repeat command. 3. If a count was applied (e.g. 3ihiEsc), the buffered text is replayed count - 1 more times so the final result is hihihi.

This is also what . (dot, repeat last change) replays for an Insert-mode change: not the keystrokes you pressed, but the text that ended up in the buffer.

Special keys pressed during Insert (Ctrl-W, Ctrl-U, Ctrl-T, Ctrl-D, Backspace, Enter) are also buffered and replayed faithfully. Insert is not just "typed text" — it is a recording of an editing action that happens to mostly be typing.

Reference

Key Action
i Insert before cursor
Esc Return to Normal mode
Ctrl-[ Return to Normal mode (alias)
Ctrl-C Return to Normal mode (cancels pending state too — but does not run autocmds; usually use Esc)

🔧 Under the Hood — Why Escape backs the cursor up one column

Leaving insert mode is not a pure mode flip. The cursor lives in a virtual column while inserting (it can sit one cell past the last character), but normal mode requires the cursor to land on a real character. So Escape clamps the column back by one when you're past the line end. That single rule is why a then Esc leaves you under the character you appended past, and why i-Esc-i doesn't drift sideways.
