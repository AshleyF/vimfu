Navigating Windows

Ctrl-W h/j/k/l — move between splits. Ctrl-W c — close. Ctrl-W o — only.

id: windows.navigate · part: windows · keys: Ctrl-W h, Ctrl-W j, Ctrl-W k, Ctrl-W l, Ctrl-W c, Ctrl-W o · lessons: 133, 134, 135

Ctrl-W with a direction key moves to the next window in that direction. Ctrl-W c closes the current window. Ctrl-W o closes everything except the current window.

Ctrl-W is the window prefix. After it, h/j/k/l jump to the window in that direction. c closes; o keeps only this one.

🔧 Under the Hood — Closing a Window vs Closing a Buffer

Ctrl-Wc (or :close) closes the window — the viewport. The buffer it was showing stays loaded; you can re-show it in another window with :b {bufname}.

:bd (:bdelete) closes the buffer — unloads the file from memory. The buffer disappears from :ls.

:q is a hybrid that confuses newcomers: it closes the current window, and if that was the last window showing the buffer, may also unload it depending on settings. Use :close when you mean window-only and :bd when you mean buffer-only.

Ctrl-Wo: "only-this-window." Closes every other window, leaving the current one full-screen. Buffers are not unloaded.

Tip. Most users remap window navigation to plain Ctrl-h/Ctrl-j/Ctrl-k/Ctrl-l (no prefix). It's the single highest-impact remap for a multi-split workflow.
