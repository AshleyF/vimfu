Creating Splits

Ctrl-W s — horizontal split. Ctrl-W v — vertical.

id: windows.splits · part: windows · keys: Ctrl-W s, Ctrl-W v, :sp, :vs · lessons: 131, 132

Ctrl-W s splits the current window horizontally; Ctrl-W v splits vertically. :sp and :vs do the same and can take a filename to open in the new split.

A window is a viewport. Splits give you more than one. Ctrl-W s stacks two viewports vertically (a horizontal split). Ctrl-W v puts them side by side (a vertical split).

🔧 Under the Hood — Splits Show Buffers; They Don't Open Files

Ctrl-Ws/Ctrl-Wv make a new window showing the current buffer. Both windows show the same file. Edit in one; the other updates immediately — same buffer.

:sp {file} (or :vs {file}) creates the split and edits a different file in the new window. Equivalent to: Ctrl-Ws then :e {file}.

This distinction (window vs buffer) is the single most important thing to internalize about Vim's window model. See `windows.buffers-vs-windows-vs-tabs`.

Tip. :new creates a horizontal split with an empty buffer. :vnew the vertical equivalent. Useful for scratch space without creating a file.
