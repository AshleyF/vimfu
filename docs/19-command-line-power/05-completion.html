Command-Line Completion

Tab to complete; Ctrl-D to list.

id: cmdline.completion · part: command-line-power · keys: Tab, Ctrl-D, Ctrl-A · lessons: 451, 451

Tab completes commands, options, filenames, and tags depending on context. Ctrl-D lists matches. Ctrl-A inserts ALL matches. The wildmenu makes Tab cycle through choices.

Vim knows what kind of argument the current command expects, and completes accordingly. Tab is context-sensitive: it completes Ex command names, option names, filenames, tag names, and more.

Key Effect
Tab Complete (or cycle, with wildmenu)
Shift-Tab Reverse-cycle through matches
Ctrl-D List all matches
Ctrl-A Insert ALL matches
Ctrl-L Complete longest unambiguous prefix
🔧 Under the Hood — What Gets Completed Where

Vim infers context from the command:

  • :e <Tab> → filenames in cwd.
  • :set <Tab> → option names.
  • :help <Tab> → help tags.
  • :colorscheme <Tab> → installed colorschemes.
  • :command <Tab> → user commands.

The 'wildmode' option controls how Tab behaves:

  • full — cycle one match per Tab.
  • longest — first complete to longest common prefix.
  • list — show all matches without completing.
  • Combined: set wildmode=longest:full,full — first Tab completes to longest unique prefix and shows menu, second cycles.

set wildmenu enables the visual completion menu — a status-line bar of choices that Tab walks through.

Tip. :e Ctrl-D is a fast way to list all files in the cwd that match what you've typed so far.
