Repeat Last Ex (@:)

@: replays your last :command. Then @@ repeats again.

id: cmdline.repeat-ex · part: command-line-power · keys: @:, @@ · lessons: 448, 448

@: replays the last Ex command. After running it once, @@ does it again. The same as macro replay — because Ex history is essentially a macro of one command.

Repeat Last Ex

Most operators have a repeat key (.). Ex commands have their own: @:.

🔧 Under the Hood — Why It's @ and Not .

. replays the last change — the last edit that modified the buffer through Normal-mode editing. Ex commands aren't part of that history; they're a separate, parallel stream.

@{reg} runs the contents of register reg as keystrokes. The colon register : always holds the last Ex command typed. So @: just runs that register — the same mechanism as macro playback (@q).

After @:, the macro-replay register becomes :, so @@ (replay last macro) runs the same Ex command again. This is why bulk operations chain naturally: :s/foo/bar/ then nnn (search next) @: @: @:.
