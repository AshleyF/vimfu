gx and g<

Open URLs and recall command output.

id: advanced.gx-g-less · part: advanced · keys: gx, g< · lessons: 493, 494

gx opens the URL or filename under cursor in the system handler (browser/etc.). g< re-displays the output of the last :command — handy when a long message scrolled past.

Two small g commands worth knowing — both solve real problems.

gx — Open URL

gx on a URL or file path passes it to the system's default opener (open on macOS, xdg-open on Linux, cmd /c start on Windows). Browsers open URLs; PDFs open in PDF readers; image files open in image viewers.

🔧 Under the Hood — How gx Decides What to Open

The implementation is in $VIMRUNTIME/plugin/netrwPlugin.vim — Vim's netrw plugin defines gx as a Normal-mode mapping that calls netrw#BrowseX(netrw#GX(), 0).

The URL/path is found by looking at the WORD under the cursor (not just word) — so URLs with ., ?, &, etc. are captured whole. The handler is selected by 'browser' if set, otherwise by platform defaults.

g< — Re-show Last Output

g< reopens the message area showing the output of the last Ex command — handy when a :registers or :set all listing scrolled past faster than you could read.

🔧 Under the Hood — Where Messages Live

Vim keeps a rolling buffer of message-area output. :messages shows the recent history; g< brings back the most recent block.

After g<, the message hit-Enter prompt appears so you can scroll the listing with Space/b, then dismiss with Enter.
