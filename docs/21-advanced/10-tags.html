Tag Navigation

Jump to definitions across files using a tags index.

id: advanced.tags · part: advanced · keys: Ctrl-], Ctrl-T, :tag, :tags, :tnext, :tprev

ctags builds a tags file mapping symbols to file:line. Ctrl-] jumps to the symbol under the cursor; Ctrl-T pops back. The whole jump history forms a stack you can browse with :tags.

Long before LSP, Vim had a 'go to definition.' It still works — and on big codebases it's faster than any language server because it's just a binary-searchable text index.

Build the index

Run a tags generator (universal-ctags is the canonical one) at the root of your project:

Command Effect
ctags -R . Recursively index everything under . into a file called tags
ctags -R --languages=python,go . Limit to specific languages
ctags -R --exclude=node_modules --exclude=.git . Skip noise

Vim looks for the tags file in the current directory, then walks parents. Configure with :set tags=./tags;/tags to search both adjacent and root tags files.

Jump and return

The tag stack

🔧 Under the Hood — Tags is a binary-searched text file

A tags file is plain text, one tag per line: NAME<TAB>FILE<TAB>SEARCH_PATTERN. The lines are sorted by NAME so the editor can find a tag with a binary search instead of a scan — which is why a 100MB tags file lookup is essentially instant. The SEARCH_PATTERN is usually a /^...$/ regex that re-finds the line in the target file, so tag jumps survive small edits to the source between regenerations.

🔧 Under the Hood — The tag stack is a real stack

Ctrl-] pushes (buffer, position, tag) onto the per-window tag stack and jumps. Ctrl-T pops. :tags prints the whole stack. Unlike the jump list (a ring buffer with duplicate suppression), the tag stack is a true LIFO — go down ten levels into a definition trail, hit Ctrl-T ten times to retrace exactly. Each window has its own stack, which is why splitting via Ctrl-W] starts a fresh trail in the new window.

Tip. Pair tags with the quickfix list (:tselect fills it with all matches) for cross-file refactor sweeps.
