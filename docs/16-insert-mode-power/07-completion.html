Word Completion (Ctrl-N / Ctrl-P)

Vim has had autocomplete since 1996.

id: insert.completion · part: insert-mode-power · keys: Ctrl-N, Ctrl-P, Ctrl-E, Ctrl-Y · lessons: 384, 388

Ctrl-N and Ctrl-P trigger keyword completion from the current and other buffers. Once a menu is up, Ctrl-N/Ctrl-P walk the suggestions; Ctrl-E cancels; Ctrl-Y accepts.

Word Completion

Type a partial word, then Ctrl-N (next) or Ctrl-P (previous) to autocomplete from words already in scope. Vim has had this since the mid-90s — long before IDE autocomplete was standard.

🔧 Under the Hood — Where Suggestions Come From

Ctrl-N/Ctrl-P use the 'complete' option (default .,w,b,u,t) to decide which sources to scan:

  • . — current buffer
  • w — buffers in other windows
  • b — other loaded buffers
  • u — unloaded buffers
  • t — tags
  • i — included files (via 'include' regex)
  • k — dictionary file

Matches are scanned in the order listed, deduplicated, and shown in a popup menu ('completeopt' controls how). The menu navigation keys (Ctrl-N/Ctrl-P/Ctrl-Y/Ctrl-E) only apply while the menu is visible.

Tip. The default just works for code: pulls from the file you're editing plus open files. For something more powerful, see Ctrl-X sub-completions next.
