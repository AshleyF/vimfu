Centering the Cursor

zz, zt, zb — move the viewport, not the cursor.

id: scroll.center-top-bottom · part: scroll · keys: zz, zt, zb · lessons: 90, 242, 243, 244

zz centers the cursor's line on screen. zt scrolls so the cursor's line is at the top. zb scrolls so the cursor's line is at the bottom.

After a search jump or function navigation, the cursor often ends up at the top or bottom edge of the visible window — too little context. Three commands fix that without moving the cursor: zz, zt, zb.

🔧 Under the Hood — These Are Inverse to H/M/L

H/M/L move the cursor to a screen position. zt/zz/zb move the screen to put the cursor at a position. They're complementary:

  • H = teleport cursor to top of screen.
  • zt = scroll the screen so the cursor (still on the same buffer line) is at the top.

Neither changes the cursor's file line — but H/M/L do change which file line the cursor is on, while zz/zt/zb do not.

A classic flow: search (/fooEnter), zz to recenter, keep reading. Or after a G, zt to push the bottom of the file to the top of the screen so you have room above the cursor.

Tip. These are part of the z prefix — see `prefixes.z-overview` for the full family. The other variants (z., z+, z-) also land the cursor on the first non-blank — see `scroll.first-non-blank-z`.

Reference

Key Action
zz Cursor's line → middle of screen
zt Cursor's line → top of screen
zb Cursor's line → bottom of screen

Worked example — zz

Bring the current line to the center.
