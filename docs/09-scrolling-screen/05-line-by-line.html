Line-by-Line Scroll

Ctrl-E and Ctrl-Y — scroll without moving the cursor (until you have to).

id: scroll.line-by-line · part: scroll · keys: Ctrl-E, Ctrl-Y

Ctrl-E scrolls the screen down one line; Ctrl-Y scrolls it up. The cursor stays on its file line until the screen would have to push it off — then it moves with the screen.

When you want to peek at what's just off-screen without moving the cursor, Ctrl-E and Ctrl-Y are the right tools. They scroll the viewport one line at a time and leave the cursor where it was — until the cursor would scroll off, in which case it slides with the screen.

Tip. Counts work: 5Ctrl-E scrolls down 5 lines. With 'scrolloff' set, the cursor will follow as needed to stay within the offset zone.
🔧 Under the Hood — Cursor Sticks Until Forced

Ctrl-E just decrements the window's visible top line. The cursor's buffer position doesn't change. Result: the screen scrolls; the cursor appears to climb upward in the visible region.

When the cursor's line is about to scroll off the top (or fall below the bottom for Ctrl-Y), Vim moves the cursor to keep it visible. So you can scroll past the cursor's line and the cursor follows along.

With 'scrolloff' > 0, the cursor moves earlier — the offset acts as a margin. Combined with 'cursorline', this is a very tactile way to read code while keeping a position marked.

Reference

Key Action
Ctrl-E Scroll screen down (1 line; counts work)
Ctrl-Y Scroll screen up
