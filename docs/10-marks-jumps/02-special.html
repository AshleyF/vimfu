Special Marks

Marks Vim sets for you automatically.

id: marks.special · part: marks · keys: '', ``, '., '^, '<, '> · lessons: 101

Some marks are set by Vim itself: '' for last jump, `` for last position, '. for last edit, '< / '> for last visual selection, and more.

Most marks you set yourself. But Vim quietly sets a handful of special marks all the time. They're worth memorizing — they make navigation feel telepathic.

Mark What it tracks
'' Position before the last big jump
`` Same, but exact position
'. Line of the last change
`. Exact position of the last change
'^ Line where Insert mode was last left
`^ Exact spot Insert was last left
'< / '> Start/end (line) of last visual selection
`< / `> Start/end (exact) of last visual selection
'[ / '] Start/end of last yanked or changed text
🔧 Under the Hood — These Power Many Familiar Commands

Special marks aren't a separate feature — they are the feature. Many commands you already use are implemented in terms of them:

  • gv re-selects the visual range stored in '< and '>.
  • gi re-enters Insert mode at `^.
  • :'<,'> as an Ex range targets the last selection.
  • `. after a jump returns you to the last edit.

Knowing the marks gives you composability the docs don't always explain. Want to yank from the start of your last yank to here? y`[. Want to delete the most recent insertion? d`[.

Tip. `` (toggle-back-to-where-you-jumped-from) is one of the most useful marks. After almost any "big" motion (G, /, %, `a), `` reverses it.
