Diff Mode

vimdiff and :diffthis — visualize and edit changes side-by-side.

id: patterns.diff · part: patterns-recipes · keys: ]c, [c, do, dp · lessons: 473, 475

Vim has a built-in diff view: vimdiff a b opens two files in vertical splits with synchronized scrolling and highlighted differences. ]c and [c jump between hunks; do and dp pull and push changes.

Vim has a real diff tool baked in. Launch with vimdiff a b from the shell, or :diffsplit other.txt to compare with the current buffer.

Key Action
]c Jump to next change
[c Jump to previous change
do Diff Obtain — pull change FROM the other file
dp Diff Put — push change TO the other file
:diffupdate Re-scan for changes after edits
:diffoff Exit diff mode for current buffer
🔧 Under the Hood — How Diff Mode Works

Diff mode is a buffer-local view setting. When you :diffthis on a buffer, Vim:

1. Marks the buffer as 'diff'. 2. Sets 'scrollbind' and 'cursorbind' so windows scroll together. 3. Sets 'foldmethod=diff' so unchanged regions fold away. 4. Runs an external diff (default: the system diff program; 'diffexpr' overrides) to find changed/added/removed lines. 5. Highlights changes with the DiffAdd, DiffChange, DiffDelete, DiffText colorgroups.

The vimdiff shell command is just vim -d — opens files in vertical splits and runs :diffthis on each.

do and dp actually edit buffers — they replace lines in one with lines from the other. Save afterwards to persist.

Tip. Comparing with the version on disk? :w !diff % - pipes the buffer to diff against the saved file.
