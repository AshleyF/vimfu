External Filter Tricks

!{motion}cmd — let any Unix tool transform your text.

id: patterns.external-filter · part: patterns-recipes · keys: ! · lessons: 476, 480

Vim's filter command !{motion} sends the text covered by motion as stdin to a shell command, replacing it with stdout. Sort, dedup, format, encode, decode — any tool works.

Vim's filter operator (Normal-mode !) sends a chunk of the buffer through a shell command and replaces it with the result. Combined with Unix tools, you have unlimited transforms.

Sequence Effect
!!sort Sort current line through sort (no-op for one line, but legal)
!ipsort Sort the inner paragraph
!Gsort Sort from cursor to end of file
:%!sort Sort the whole file
:%!sort -u Sort + dedup
:%!jq . Pretty-print JSON in place
:%!python -m json.tool Same, with Python
:%!xmllint --format - Pretty-print XML
:%!fmt -w 80 Reflow to 80-column lines
:'<,'>!base64 Base64-encode the visual selection
🔧 Under the Hood — How Filter Mode Works

From Normal mode, !{motion} prompts for a shell command on the command line. When you press Enter:

1. The lines covered by motion are written to a temp file. 2. The shell command is run with that file as stdin. 3. The motion's lines are replaced in the buffer with the command's stdout.

!! (doubled bang) targets the current line. From Ex, :[range]!cmd does the same with an explicit range.

The filter is line-oriented even from Normal mode — !iw doesn't filter just a word; the entire line(s) containing the word are sent. (Use the substitute command for sub-line transforms.)

Tip. Make a habit of using filters for one-shot needs (sort, format, encode). It's faster than alt-tabbing to a shell and back.
