gv — Re-Select

Bring back the last visual selection.

id: visual.gv · part: visual-depth · keys: gv · lessons: 444, 445

gv re-enters Visual mode with the same selection you last had — same shape, same endpoints. Essential for re-applying an operator to the same region.

gv re-enters Visual mode with the exact same selection you last had — same shape, same lines, same characters.

🔧 Under the Hood — Where the Selection Is Stored

gv reads the special marks `< and `> (start and end of last visual) plus an internal record of the visual mode shape (character/line/block) that was active. Together those three things are enough to recreate any visual selection.

The selection persists across:

  • Mode changes (Normal/Insert/Ex)
  • Cursor moves
  • Most edits — though if your edit removes the selected text, gv may select an empty or unexpected region.

gv doesn't survive :e (re-edit). The marks are buffer-local: each buffer has its own last-visual.

Tip. After a :s substitution that you ran on a visual selection, gv brings the (modified) selection back so you can re-substitute or re-indent.
