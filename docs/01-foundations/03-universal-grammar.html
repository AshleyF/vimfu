The Universal Grammar

[count] [register] operator [count] motion

id: foundations.universal-grammar · part: foundations · keys: d, c, y, >, <, =, gU, gu, g~, w, iw, ip, $, % · lessons: 85

Vim is a tiny language: verbs (operators) compose with nouns (motions and text objects). Once you see the grammar, every key falls into place.

Almost every command in Vim follows one shape:

Read it as a sentence: do this thing, optionally to register r, this many times, to this stretch of text. Concrete examples:

Command Reads as
dw Delete one word forward
3dw Delete three words forward
d3w Delete three words forward (counts multiply, but here one of them is 1)
ciw Change the inner word the cursor is on
y$ Yank from cursor to end of line
>ip Indent inner paragraph
"ayy Yank this line into register a
gUiw Uppercase the inner word

The verbs (operators)

Operator Meaning
d Delete (cut)

> Indent right
< Indent left

gU Uppercase
gu Lowercase
g~ Toggle case
gq Format (line-wrap)
! Filter through external command

The nouns (motions and text objects)

Anything that moves the cursor in Normal mode is a motion. After an operator, the same motion describes the range the operator acts on. So w alone moves the cursor a word forward; dw deletes the same span.

Noun kind Examples
Word / WORD w, b, e, W, B, E
Line 0, ^, $, _, g_
File gg, G, {nG}
Find on line f{char}, F{char}, t{char}, T{char}, ;, ,
Search /pattern, ?pattern, n, N, *, #
Sentence / paragraph (, ), {, {key:}}
Bracket match %
Inner / around text object iw, aw, i", a(, ip, at
🔧 Under the Hood — Why It Composes

Operators and motions live behind a clean interface. A motion is a function cursor → (start, end, type), where type is char, line, or block. A text object is a function cursor → (start, end, type) that doesn't have to use the cursor as one of the endpoints. An operator is a function (start, end, type) → effect-on-buffer.

This is why every operator works with every motion and every text object. Vim's source code does not have a giant switch statement of "if d, then handle w specially." It has one operator dispatch and one motion dispatch, and they meet through the (start, end, type) triple. When you learn a new operator, you've learned its combination with every motion you already know. When you learn a new text object, you've learned its combination with every operator. The grammar scales multiplicatively.

🔧 Under the Hood — The Doubled Form

Every operator has a doubled form that means "apply linewise to this line": dd deletes the line, yy yanks the line, cc changes the line, >> indents the line, gUgU (or gUU) uppercases the line. This is a syntactic shortcut — internally, dd is processed as a special case where the motion is replaced by "the current line, linewise."

🔧 Under the Hood — The Wart: Y, D, C, S

Vi shipped before the doubled-form rule was systematic. Y is not y$ — historically it was yy (yank line). Modern Neovim has finally fixed this default, but vanilla Vim still aliases Y to yy. D, on the other hand, is d$ — it has always been the consistent form. C is c$. S is cc. s is cl. These shortcuts predate the unified grammar and are kept for muscle memory. Treat them as exceptions to learn, not principles to derive.

Tip. When you want to learn a new key, ask: is it a verb (operator), a noun (motion / text object), or a complete command (like x or p)? If it's the first two, you've immediately doubled the size of your vocabulary by combining it with everything you already know.
