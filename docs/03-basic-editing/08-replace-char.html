Replace a Character

r{char} — overwrite one character without entering Insert mode.

id: editing.replace-char · part: editing · keys: r · lessons: 45

r{char} replaces the character under the cursor with {char} and stays in Normal mode. Perfect for fixing a typo without an i…Esc dance.

Typo? Cursor on the wrong letter? r{c} swaps it for {c} and leaves you in Normal mode. No Insert-mode round-trip, no Esc. One change, one keystroke past the new character.

Tip. rEnter replaces the character with a newline — a sneaky way to break a line at the cursor without entering Insert mode.
🔧 Under the Hood — Why r Doesn't Enter Insert Mode

r is a one-shot operation: it consumes exactly one keystroke as its argument and applies it as a literal replacement. Internally it's not the same as s (substitute), which deletes the character and switches to Insert mode. r stays in Normal mode the entire time and produces a single change in the undo tree.

With a count, {n}r{c} replaces {n} characters with the same {c} repeated. Useful for -------- style separators.

Tip. If you need to overwrite many characters with different ones, use Replace mode (R) — it's like Insert mode but each typed character overwrites instead of pushing. See `transform.replace-mode`.

Reference

Key Action
r{c} Replace one character with c
{n}r{c} Replace next n characters with c (repeated)
rEnter Replace with a newline (split line)
R Enter Replace mode (continuous overwrite)
gr{c} Replace virtually (don't extend short lines)

🔧 Under the Hood — Replace mode keeps a shadow buffer

R can't be implemented as 'overwrite forward' alone, because Backspace must restore each original character. So replace mode tracks a small shadow region — the line as it was at entry — and the Backspace handler diffs against that shadow to put back what you replaced. The shadow also clamps how far Backspace can go: you can never erase past your entry point.
