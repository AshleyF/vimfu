Line Motions

0 / ^ / $ — the three places that matter on a line.

id: editing.line-motions · part: editing · keys: 0, ^, $ · lessons: 25, 26, 27

0 jumps to column 0. ^ jumps to the first non-blank character. $ jumps to the end of the line. They're motions, so d$ deletes to end of line, c0 changes to start.

A line has three interesting positions: the actual start (column 0), the first non-blank character (where the code begins), and the end. Vim gives them three keys.

Tip. ^ is what you almost always want. 0 is for when the leading whitespace itself is the target — for example, d0 to delete the indentation.
🔧 Under the Hood — Inclusive vs Exclusive — Why $ Behaves Subtly

Motions in Vim have a type that decides what an operator does with them:

  • Exclusive motions don't include the character under the destination cursor. w is exclusive: dw deletes up to but not including the next word's first character.
  • Inclusive motions do include it. e is inclusive: de deletes through the end of the word.
  • Linewise motions act on whole lines. j is linewise: dj deletes two whole lines.

0 is exclusive; ^ is exclusive; $ is inclusive — that's why d$ deletes through end-of-line including the last character. The Universal Grammar (operator + motion) keeps working, but the precise byte ranges depend on the motion's type. Full classification: `appendix.motion-classification`.

Tip. There's a shortcut: D = d$ (delete to end of line). C = c$. Y should be y$ but historically isn't — see the wart in `editing.yank-put`.

Reference

Key Lands on Type
0 Column 0 (the actual start) Exclusive
^ First non-blank character Exclusive
$ End of line Inclusive
g_ Last non-blank character Inclusive

🔧 Under the Hood — Blank vs whitespace-only is a real distinction

First-non-blank motions (^, _, +, -) and screen motions (H, M, L) all hinge on two predicates the editor evaluates per line: length is zero (blank) and first non-whitespace column. A whitespace-only line is not blank — it counts as having content for paragraph motions but yields a no-op for ^. Holding both predicates straight is the only way these motions feel consistent.
