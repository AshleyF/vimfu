The [ and ] Family

[ goes back, ] goes forward. The semantic siblings of search.

id: prefixes.bracket-pairs · part: prefixes · keys: [(, ]), [{, ]}, [m, ]m, [c, ]c, [s, ]s · lessons: 317, 333

[ and ] form a paired-prefix family for navigating semantically: previous/next bracket, function, comment, change, spelling error, etc. Mostly used in code.

The bracket-prefix family is symmetric: [ always means "backward," ] always means "forward." The second key picks the thing to look for. [c = previous diff hunk; ]c = next diff hunk. [m = previous method start; ]m = next.

🔧 Under the Hood — Symmetry Is the Whole Idea

Once you internalize the convention, the entire family becomes guessable:

  • [( / ]) — previous / next unmatched paren.
  • [{ / ]{key:}} — previous / next unmatched brace.
  • [m / ]m — previous / next method start.
  • [M / ]M — previous / next method end.
  • [# / ]# — previous / next preprocessor #if/#endif.
  • [s / ]s — previous / next spelling error.
  • [c / ]c — previous / next diff change.
  • [I — list lines containing the keyword under cursor (powerful, obscure).

Most are language-aware via filetype plugins. [m on a Java buffer finds method starts; on plain text it does nothing useful.

Reference

Key Action
[( / ]) Previous / next unmatched (
[{ / ]{key:}} Previous / next unmatched {
[m / ]m Previous / next method start
[M / ]M Previous / next method end
[c / ]c Previous / next diff change
[s / ]s Previous / next spelling error
[I List occurrences of keyword under cursor
[p / ]p Put before / after, adjusting indent
[# / ]# Previous / next preprocessor directive
🔧 Under the Hood — Bracket-prefix commands are 'navigate by structure'

The [ and ] prefixes share a theme: jump to the next/previous structural token. [[, ]] jump by section; [m, ]m by method; [s, ]s by misspelled word; [c, ]c by diff hunk. Each pair shares the same direction-flip implementation; only the structural predicate changes.
