Repeat Last Change

. — replay the last change. The single most useful key.

id: dot.repeat · part: dot · keys: . · lessons: 71

The . command repeats the last change — exactly what you did, including any inserted text. It is what makes Vim feel like Vim.

If Vim has a single most-loved key, this is it. . replays the last change. Not the last keystroke, not the last motion — the last change. That distinction is the whole reason . feels magical.

🔧 Under the Hood — What Counts as "a Change"

Vim has a precise definition of change (and only changes can be dotted):

  • An entire Insert-mode session, from i/a/o/etc. through Esc, is one change. The text you typed is part of it.
  • Every operator+motion (dw, cip, >ip) is one change.
  • Every single-shot edit (x, r{c}, p, ~) is one change.
  • Pure motions and searches are not changes. j, G, /, n don't update the dot register.

Vim records the commands and any typed text of the last change in the dot register (".). Pressing . replays them at the current cursor position.

So cw-type-hello-Esc is one change. . on a different word will change-word-to-hello again. o-type-a-line-Esc is one change. . opens another line and types the same content.

Tip. Search (/) is not a change, but it pairs with . beautifully: /fooEnter-cw-bar-Esc → n. n. n. walks you through every match. Find next; repeat the change. Two keys to refactor each occurrence.
Tip. u undoes one change. . repeats one change. Together they let you preview-and-undo refactors: do it once, see if you like it, u if not, n. if yes. Confidence at speed.

Reference

Key Action
. Repeat last change at cursor
{n}. Repeat last change with new count {n}
".p Paste the actual recorded change as text
