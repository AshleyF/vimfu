Named Registers

"a–"z — your 26 personal slots. Capitals append.

id: registers.named · part: registers · keys: "a, "z, "A, "Z · lessons: 119, 120

Lowercase "a–"z are 26 named registers you write to explicitly. Uppercase "A–"Z append to the same registers instead of overwriting.

26 lowercase letters give you 26 named registers. Pick one with "{letter} before an operator: "ayy yanks the line into register a. "ap pastes it back.

🔧 Under the Hood — Capitals Append

"ayy overwrites register a with the current line.

"Ayy appends the current line to register a (preserving what was already there).

This is one of Vim's small genius moves: collecting a list of items into one register without an external paste buffer.

Classic use: scan a file, find every line matching TODO, append each to register t, then dump them somewhere with "tp.

With :g/{pat}/y T, the global command does the same thing: yank each matching line appending to register t. Then put.

🔧 Under the Hood — Same Registers Hold Macros

When you record a macro with qa…q, those keystrokes go into register a — exactly the same a you yank text into. Macros are just register contents.

This means you can:

  • Yank a sequence of keystrokes you typed into a buffer with "ay, then play it as a macro with @a.
  • Edit a macro by pasting it into the buffer ("ap), tweaking, then yanking back ("ay$). See `macros.editing`.
  • Compose macros: "Ayy appends a line to a recorded macro register, extending what @a will replay.

This unification is one of the most beautiful things in Vim's design.

🔧 Under the Hood — Named registers are explicit destinations

Selecting a register with "a routes the next command's output (or input, for paste) through that slot — once. The selection is consumed by the next operator, then resets. You can't 'set' a register persistently in normal mode; you can only direct individual commands at one. Capital letters ("A) append instead of overwrite.
