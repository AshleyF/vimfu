System Clipboard Registers

"+ and "* — bridge to the OS clipboard.

id: registers.clipboard · part: registers · keys: "+, "* · lessons: 124

"+ is the system clipboard. "* is the X11 selection (Linux) or another alias for the clipboard. Yank into them, paste from them, or set 'clipboard' to make Vim use them by default.

If you want text from Vim to land in your system clipboard — or vice versa — use "+. "+yy yanks a line into the clipboard. "+p pastes from it.

🔧 Under the Hood — + vs *

Two clipboard registers exist on Linux/X11:

  • "+ — the CLIPBOARD selection. The thing Ctrl-C/Ctrl-V uses in most apps. Use this one by default.
  • **"* — the PRIMARY** selection (X11). The thing middle-click pastes. Less universal; only matters in X11.

On macOS and Windows, both "+ and "* map to the same system clipboard. On WSL, you may need extra setup (e.g. xclip, win32yank, or Neovim's clipboard provider).

If Vim was built without +clipboard, neither register exists. Check with :version; look for +clipboard. Neovim ships with clipboard support that works wherever a provider exists.

🔧 Under the Hood — Make Yank/Paste Use the Clipboard by Default

Add to your config:

set clipboard=unnamedplus

Now y writes to "+ (and ""); p reads from "+. No more "+ prefix. The downside: you lose the ability to yank "just to Vim" without touching the system clipboard. Most people find that's a fine trade.

On macOS without X11, use clipboard=unnamed instead (writes to "*, which equals the system clipboard there).
