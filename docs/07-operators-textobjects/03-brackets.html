Bracket Text Objects

i( i{ i[ i< — operate on the contents of any bracket pair.

id: operators-textobjects.brackets · part: operators-textobjects · keys: i(, a(, i{, a{, i[, a[, i<, a<, ib, iB · lessons: 78, 79, 80, 81

i( / a( for parens, i{ / a{ for braces, i[ / a[ for square brackets, i< / a< for angle brackets. ib and iB are aliases for parens and braces.

Bracket objects are the most-used text objects in real code. They're how you yank arguments, change function bodies, and delete array literals without ever counting characters.

Tip. ib = i(. iB = i{. The letter aliases exist because some keyboards make ( and { painful to type repeatedly. Pick whichever you prefer.
🔧 Under the Hood — How Bracket Matching Works

Bracket objects walk outward from the cursor looking for an unbalanced opener of the requested type. They respect nesting — i( on the inner of f(g(x)) selects g(x); from outside the inner parens it selects g(x) if you're inside the outer pair.

Matching uses the 'matchpairs' option (default (:),{:},[:],<:>). It does not understand language syntax — strings and comments don't shield brackets, which is occasionally surprising in code with mismatched brackets in comments.

Bracket objects are charwise. Even i{ on a multi-line block selects character-by-character — meaning the leading and trailing newlines are part of the range. That's usually what you want for change/delete, but it surprises people the first time yi{ pastes back as a multi-line span starting in the middle of a line.

Tip. Pair these with the grammar's other halves: >i{ indents inside braces, =a( re-indents the entire parenthesised expression, gUi[ uppercases everything inside the brackets.
