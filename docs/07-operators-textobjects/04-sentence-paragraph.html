Sentence and Paragraph

is / as / ip / ap — for prose.

id: operators-textobjects.sentence-paragraph · part: operators-textobjects · keys: is, as, ip, ap · lessons: 82, 83

is is the sentence containing the cursor. ip is the paragraph. The 'a' variants include trailing whitespace.

Most of Vim's text objects are for code. These two are for prose — sentences and paragraphs as units of text.

🔧 Under the Hood — What Vim Means by Sentence and Paragraph

Sentence: ends at ., !, or ? followed by spaces or end-of-line. Some closing punctuation (), ], ", ') may follow before the spaces. This is set via the 'joinspaces' and a few related options.

This definition is about typed text, not English semantics. "Mr. Smith." is two sentences to Vim. Lists with 1. are confusing too. It's a heuristic; for code or markdown lists it's often wrong, but for paragraphs of plain prose it works.

Paragraph: a run of non-empty lines, separated by empty lines. Some configurations also count nroff-style macros as paragraph separators (the 'paragraphs' option), but the default — blank-line separation — is what you'll usually see.

Tip. gqip reformats the current paragraph to fit 'textwidth'. Combined with markdown-mode it's how many writers bulk-tidy a chapter.

Reference

Object Range
is Inner sentence (no trailing space)
as Sentence + trailing whitespace
ip Inner paragraph (no surrounding blank lines)
ap Paragraph + trailing blank line
