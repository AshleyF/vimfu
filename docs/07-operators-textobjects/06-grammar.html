The Vim Grammar

[count] [register] operator [count] {motion | text-object}

id: operators-textobjects.grammar · part: operators-textobjects · keys: [count], [reg], op, motion, text-object · lessons: 85

Every editing command in Vim is one sentence: optional count, optional register, an operator (verb), then a motion or text object (noun). This is the grammar that makes Vim feel infinite from a small number of keys.

Most editors have commands: "delete word," "select to end of line," "comment block." Vim doesn't. Vim has a grammar — a small set of verbs and nouns that you compose into sentences. Once you see it, you stop memorizing keys.

🔧 Under the Hood — The Grammar in One Line

Every editing action in Normal mode follows the form:

[count] ["register] operator [count] {motion | text-object}

All the brackets are optional. Concrete examples:

  • dw — delete one word (no count, default register, motion).
  • 5dw — delete five words.
  • "a5dw — delete five words into register a.
  • cip — change the inner paragraph (text object).
  • >a{ — indent the brace block including its braces.
  • gUtZ — uppercase up to the next Z.

There is no command list to memorize. There are operators. There are motions and text objects. Multiplication is the count. The register is where the result goes.

🔧 Under the Hood — The Operators

The full operator set (the verbs):

  • d — delete (cut into a register).
  • c — change (delete + Insert).
  • y — yank (copy).
  • > < — indent / dedent.
  • = — re-indent (use language indenter).
  • gu gU g~ — lowercase / uppercase / toggle case.
  • gq — reformat (text-wrap).
  • ! — filter through external command.
  • g@ — call user-defined operatorfunc (advanced).

That's it. ~10 verbs. Each one takes any motion or text object. Pair them up: that's the language.

🔧 Under the Hood — Doubled Operators

Every operator has a doubled form that means "act on the current line":

  • dd — delete line.
  • cc — change line.
  • yy — yank line.
  • >> << — indent / dedent line.
  • == — re-indent line.
  • gUU or gUu — uppercase line.

This is a special case in the grammar, not a separate command. The pattern is the rule: doubled operator = current line.

🔧 Under the Hood — Synonyms (Single-Key Aliases)

Some patterns are common enough to deserve a one-key shortcut:

  • D = d$
  • C = c$
  • Y = yy (classic vi) / y$ (Neovim default)
  • S = cc
  • s = cl
  • x = dl
  • X = dh

These are sugar over the grammar. Knowing both forms helps when reading other people's keystrokes — and explains why Y is a wart (broken symmetry, see `editing.yank-put`).

Tip. If you can answer "what's the verb? what's the noun?" for a command you want to run, you can usually type it. That's the test for whether you've internalized the grammar — and once you have, every new motion or text object is a free upgrade for every operator.
