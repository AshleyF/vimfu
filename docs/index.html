VimFu

The Vim & Neovim reference for programmers — companion to the book.

This is the online reference half of VimFu. Quick to scan, deep-linkable, packed with embedded videos. Every topic in the book has a page here so you can look something up without flipping pages.

The printed book goes deeper — history, internals, mental models, and worked examples for every concept. The site is the cheat sheet; the book is the why.

Basic Editing

  • More Ways into Insert — Five doors into Insert mode — pick the one that lands the cursor where you want.
  • Word Motions — Move a word at a time. The grammar's first real noun.
  • Line Motions — 0 / ^ / $ — the three places that matter on a line.
  • File Motions — gg, G, {n}G — jump anywhere in the file.
  • Delete — d is the verb. Pair it with any motion.
  • Change — c is delete-then-insert, atomically.
  • Yank and Put — Yank is copy. Put is paste. The register knows whether it's a line or a span.

Search Find

  • Pattern Search — / searches forward, ? searches back. n and N step through matches.
  • Word Search — * / # — search for the word under the cursor without typing it.
  • Find on Line — f, F, t, T — jump to a character without leaving the line.

Counts Visual

  • Counts — Type a number before any motion or operator. It multiplies.
  • Percentage and Column Jumps — {n}% jumps to a percentage of the file. {n}| jumps to a column.
  • Character Visual Mode — v — select first, act second.
  • Line and Block Visual — V selects whole lines. Ctrl-V selects rectangles.
  • Visual Mode Tricks — Block O for diagonal corners. gv to re-select.

Dot Repeat

  • Repeat Last Change — . — replay the last change. The single most useful key.
  • Dot with Counts — {n}. — replay the change with a new count.
  • Dot Patterns — Search + change + dot. The two main refactor idioms.

Operators Textobjects

  • Inner Word vs A Word — iw and aw — the first text objects.
  • Quote Text Objects — i" / a" / i' / a' — operate on a quoted string without finding its boundaries.
  • Bracket Text Objects — i( i{ i[ i< — operate on the contents of any bracket pair.
  • Sentence and Paragraph — is / as / ip / ap — for prose.
  • HTML/XML Tag Objects — it / at — operate inside an HTML or XML tag pair.
  • The Vim Grammar — [count] [register] operator [count] {motion | text-object}

Wider Motions

  • Sentence and Paragraph Motions — ( ) — sentences. { } — paragraphs.
  • Match Bracket — % — jump to the matching paren, brace, or bracket.
  • Screen Position Jumps — H, M, L — High, Middle, Low on the visible screen.
  • Line-Start Motions — +, -, Enter — jump to first non-blank of next/previous line.
  • WORD Motions — W, B, E, gE — like w/b/e/ge but bigger words.

Scrolling Screen

  • Half- and Full-Page Scrolling — Ctrl-D / Ctrl-U / Ctrl-F / Ctrl-B — scroll without losing place.
  • Centering the Cursor — zz, zt, zb — move the viewport, not the cursor.
  • z. / z+ / z- — Scroll and Land on First Non-Blank — Like zz/zt/zb, but the cursor jumps to the first non-blank.
  • z{n}{Enter} — Set Window Height — Resize the current window to {n} lines, scroll cursor's line to the top.
  • Line-by-Line Scroll — Ctrl-E and Ctrl-Y — scroll without moving the cursor (until you have to).
  • Horizontal Scroll — zs, ze, zh, zl — scroll left/right when wrap is off.

Marks Jumps

  • Setting and Jumping to Marks — m{a-z} sets a mark. '{a-z} jumps to its line. `{a-z} jumps to its exact spot.
  • Special Marks — Marks Vim sets for you automatically.
  • Tick vs Backtick — ' jumps to the line of a mark. ` jumps to the exact position.
  • The Jump List — Ctrl-O / Ctrl-I — walk back and forward through your big jumps.
  • The Change List — g; / g, — walk through the places you've been editing.

Transform

  • Replace Mode — R — overwrite as you type. r{c} — overwrite one character.
  • Indent and Auto-Indent — >> indents. << dedents. ={motion} re-indents using the language indenter.
  • Case Changing — ~ flips one character. gU/gu/g~ are operators.
  • Joining Lines — J joins the next line to the current. gJ joins without a space.

Registers

  • Registers — A dictionary of named clipboards built from one primitive
  • The Unnamed Register — "" — where every yank and delete lands by default.
  • Named Registers — "a–"z — your 26 personal slots. Capitals append.
  • Numbered Registers — "0 = last yank. "1–"9 = the last nine deletes.
  • The Small Delete Register — "- — for charwise deletes that wouldn't be worth a numbered slot.
  • System Clipboard Registers — "+ and "* — bridge to the OS clipboard.
  • Special Registers — "/ ": ". "% "# "= "_ — the read-only and weird ones.
  • Macros Are Just Register Contents — "ap → edit → "ay$ — the macro lifecycle as text editing.

Macros

  • Recording and Playing Macros — qa records into register a. q stops. @a plays.
  • Counted Macros — 100@a — run a macro a hundred times. Or until it errors out.
  • Editing a Macro — Paste it, fix it, yank it back. Macros are text.

Windows Buffers Tabs

  • Buffers — Files in memory, navigated with :ls and :b.
  • Tabs — :tabnew, gt, gT — tabs are layouts of windows, not files.
  • Buffer vs Window vs Tab — Three concepts most editors collapse into one. Vim keeps them separate.
