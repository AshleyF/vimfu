Visual Mode Tricks

Block O for diagonal corners. gv to re-select.

id: counts-visual.tricks · part: counts-visual · keys: O, gv · lessons: 69, 70

Two tricks that make Visual mode much more useful: O in block mode swaps to the opposite diagonal corner, and gv re-selects the previous selection.

Two small tricks save lots of pain in Visual mode: O (capital O) for moving across the diagonal of a block, and gv for getting back the selection you just lost.

O — diagonal corners

In char/line Visual, o swaps which end of the selection your motions move. In block Visual, o swaps along one diagonal — but O (capital) swaps to the opposite corner of the rectangle. So you can extend the block from any of the four corners without restarting.

gv — re-select last visual

Pressed Esc by accident? Did one operation and want to do another on the same range? gv restores the last visual selection — same anchor, same cursor, same mode.

🔧 Under the Hood — How gv Works

Whenever you leave Visual mode, Vim writes two marks (`<, `>) and remembers the type of the selection. gv re-enters that type of Visual with the cursor and anchor restored from those marks.

Because it's just two marks, anything that updates them updates gv's target. :s commands run on the visual range update them implicitly — so you can :'<,'>s/foo/bar/g and then gv to grab the same range again.

The range works per buffer. Open a different buffer; come back; gv still works.

Reference

Key Action
o Swap cursor / anchor (any visual)
O Move to opposite corner (block visual only)
gv Re-enter previous visual selection
🔧 Under the Hood — o swaps the anchor, not the text

In visual mode, o (and O for visual-block) flips which endpoint is the anchor and which is the cursor. The selection contents don't change — only which end your motions extend. After an operator runs, the cursor lands on whichever endpoint was current at consume time, so swapping ends before d or y changes where you end up.
