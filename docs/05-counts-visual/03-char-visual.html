Character Visual Mode

v — select first, act second.

id: counts-visual.char-visual · part: counts-visual · keys: v, o · lessons: 64, 68

v enters character-wise Visual mode. Move to extend the selection. Press an operator (d, c, y, …) to act on it.

Visual mode flips the grammar around. Instead of "verb then noun" (dw), you do "noun then verb": select first, then act. v starts the selection, motions extend it, and an operator finishes the job.

Tip. Press Esc or v again to leave Visual mode without acting. Press an operator to act and return to Normal.
🔧 Under the Hood — Visual Is Just an Anchored Cursor

When you press v, Vim sets an anchor at the cursor and remembers it. Every motion now updates the cursor while the anchor stays put. The selection is the (anchor → cursor) range — and Vim keeps it sorted so the lower position is always the start.

o swaps the anchor and cursor. So if you started a selection going forward and realize you need to extend it backward, o flips which end your motions move.

When you finally press an operator, Vim hands the operator the (start, end) pair just like a motion would. Visual mode and operator+motion are the same thing internally — they just package the range differently.

🔧 Under the Hood — The Last Visual Selection Is Remembered

Whenever you exit Visual mode, Vim sets two automatic marks: `< (start) and `> (end). They live in the buffer until your next visual selection.

This means:

  • gv re-selects the last visual range.
  • :'<,'> (or just : from Visual) targets it as an Ex range — useful for :s/foo/bar/g.

Most "do an Ex command on what I just selected" idioms work this way. See `visual.range`.

🔧 Under the Hood — Visual mode is anchor + endpoint

Charwise visual mode stores two positions: the anchor (where you pressed v) and the current cursor. Motions move the cursor; the selection is always the inclusive range between the two. There's no selection object being copied around — the renderer paints whatever sits between anchor and cursor each frame. Operators consume the resolved range and exit back to normal mode.
