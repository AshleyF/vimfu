Find on Line

f, F, t, T — jump to a character without leaving the line.

id: search.find-on-line · part: search · keys: f, F, t, T, ;, , · lessons: 52, 53, 54, 55, 56, 57

f{c} jumps to the next {c} on the line; F{c} jumps to the previous one. t{c} stops one character before; T{c} stops one character after. ; and , repeat in the same and opposite directions.

/ is overkill for jumping to a paren on the same line. f{c} jumps to the next {c} without ever leaving the line. F jumps backward. t ("till") stops one short. Then ; repeats the jump.

Tip. t is the operator-friendly variant. dt) deletes everything up to the next ) without eating it. df) eats it too. Choose based on whether you want to keep the delimiter.
🔧 Under the Hood — Why ; Is So Useful

f, F, t, T all stash two pieces of state: the last character you searched for, and the direction (or modality — t vs f). ; replays both. , replays the character but flips the direction.

So f,-then-jump-to-some-other-thing-then-; re-finds the next comma. You don't have to retype it; the keyboard remembers. This is one of those rare cases where two pieces of remembered state make a one-key motion enormously useful.

This state is per-window, not per-buffer. Switching files keeps your last find.

Tip. f only sees the current line — it never crosses a newline. For cross-line jumps, use search (/).

Reference

Key Lands on Type
f{c} Next {c} on line Inclusive
F{c} Previous {c} on line Exclusive
t{c} Char before next {c} Inclusive
T{c} Char after previous {c} Exclusive
; Repeat last find (same direction) Same as original
, Repeat last find (opposite) —
