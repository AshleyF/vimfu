Pattern Search

/ searches forward, ? searches back. n and N step through matches.

id: search.pattern · part: search · keys: /, ?, n, N · lessons: 46, 47, 48, 49

/pattern searches forward; ?pattern searches backward. n repeats the search in the original direction; N reverses it. Search is one of Vim's most powerful motions.

Vim's search is two-stage: type / (or ?) at the cursor's current line, type the pattern, press Enter. Cursor jumps to the first match. n for the next match, N for the previous.

Tip. The pattern is a regex, not a literal string. /foo.Enter matches foo followed by any character. To search for a literal ., escape it: /foo\.Enter. Or use the \V (very nomagic) prefix to disable regex meaning altogether: /\Vfoo.Enter.
🔧 Under the Hood — Search Is a Motion

/{pattern}Enter is a motion: it produces a (start, end) pair just like w or G. So every operator pairs with it.

  • d/ENDEnter — delete from here through the next END.
  • c/fnEnter — change from here to the next fn.
  • y?defEnter — yank from here back to the previous def.

Search is also a jump — it pushes the previous position to the jumplist, so Ctrl-O returns you. And the pattern itself is stored in the / register ("/), reusable in :s commands.

🔧 Under the Hood — Settings That Change How Search Feels
  • 'incsearch' — show matches as you type. On by default in Neovim.
  • 'hlsearch' — highlight all matches after Enter. :nohEnter clears the highlight without losing the search history.
  • 'ignorecase' — case-insensitive search.
  • 'smartcase' — combined with ignorecase, search is case-insensitive unless you typed a capital letter. The pragmatic default.
  • 'wrapscan' — when the search hits the end of file, wrap to the top. On by default; turn off if you want hard stops.

The pattern syntax has four "magic" levels: \v (very magic — most regex chars are special), default \m, \M (nomagic), \V (very nomagic — almost everything literal). For complex patterns prefer \v.

Tip. Empty pattern reuses the previous one: /Enter jumps to the next match of whatever you last searched for.

Reference

Key Action
/{pattern}Enter Forward search
?{pattern}Enter Backward search
n / N Next / previous (in original direction)
/Enter Repeat last search forward
?Enter Repeat last search backward
