Word Search

* / # — search for the word under the cursor without typing it.

id: search.word · part: search · keys: *, #, g*, g# · lessons: 50, 51

Put the cursor on a word, press to search forward for the next occurrence, # for the previous. g and g# do the same but without word boundaries.

Cursor on a word, press *, jump to the next occurrence. Press #, jump to the previous. No typing the pattern, no closing Enter.

🔧 Under the Hood — What * Actually Does

* is not "search for the literal text under the cursor." It's: take the keyword under the cursor, wrap it in \<…\> (whole-word boundaries), and run that as a forward search.

So with the cursor on foo in foo barfoo bazfoo, * only matches the standalone foos, not the foo inside barfoo. That's almost always what you want.

When it isn't — for example, refactoring an identifier where you also want to catch getFoo and Foobar — use g* (no boundaries). It builds the pattern without \<…\>.

After *, the search is in the / register, so you can edit it: q/ opens the search history window and lets you tweak the pattern.

Tip. * is the start of one of the most useful refactor patterns: *-cgn-type-replacement-Esc-.-.-… Walk through every match, change-or-skip with n or .. See `recipes.cgn-dot`.

🔧 Under the Hood — * synthesizes a regex from the cursor token

Word search is a tiny code generator. It tokenizes the text under the cursor, wraps it in word-boundary anchors (\<...\>), and writes the result into the search-pattern slot. # is the same generator with reversed initial direction. The g-prefixed variants (g*, g#) drop the boundary anchors. Knowing this, you can predict exactly which token will be selected and why partial matches behave as they do.
